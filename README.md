# A4仕上がり 折りパンフレットシミュレーター

折りたたむと A4(210×297mm)になる二つ折り・三つ折りパンフレットの
仕上がりイメージを、ブラウザ上で3Dプレビューできるツールです。
展開サイズは三つ折りで約630×297mm、二つ折りで約420×297mm(A3相当)。
すべてクライアントサイド(Three.js)で動作し、アップロードした画像はサーバーに送信されません。

## 機能

| 機能 | 内容 |
|---|---|
| 折り方選択 | 巻き三つ折り(C折り)/ Z折り / 二つ折り |
| 表紙位置 | 右が表紙(標準)/ 左が表紙(逆折り) |
| デザインアップロード | 表面(OUTER)・中面(INNER)の各パネル(面)ごとに1枚ずつ画像を割り当て。各スロットへドラッグ&ドロップ、またはクリックで選択。パネル数は折り方に追従(三つ折り=各3面/二つ折り=各2面)。各スロットの×ボタンで画像を削除(ガイドに戻る) |
| 折り具合スライダー | 0〜100%(初期値 0%) |
| 照明の明るさ | 0〜100%(初期値 75%) |
| プレビュー上下反転 | 描画結果を縦ミラー反転(ドラッグの縦操作も反転補正) |
| 両面を画像で保存 | 展開状態の表面・中面を上下に並べた1枚のPNGプルーフを書き出し |
| パンフレット依頼 | 画面右下のボタンで依頼フォーム(モーダル)を表示。基本情報・依頼内容を確認して送信すると、両面プルーフ画像を自動添付して担当者メールへ通知が届く(FormSubmit 利用) |
| 3D操作 | ドラッグで回転、ホイール / ピンチで拡大縮小、視点リセットボタン |

## 使い方

`index.html` をブラウザで開くだけで動作します(要インターネット接続:Three.js をCDNから読み込み)。

ローカルサーバーで配信する場合:

```
npx serve trifold-simulator
```

## 実装メモ

- 1ファイル構成(`index.html`)。依存は Three.js r128(CDN)のみ。
- 用紙は薄い `BoxGeometry`(各210×297×0.6mm、1unit=1mm)のパネルで構成。折り方に応じて
  `buildPaper()` がパネル数を組み替える(三つ折り=3枚/二つ折り=2枚)。各パネルはヒンジ用の
  `Group`(pivot)の子にして回転させる。三つ折りは中央パネル固定、二つ折りは中央の折り線を軸にする。
- 巻き三つ折りは「反対側パネルを先に折り込み(スライダー前半)→表紙側パネルで外から巻く(後半)」の2段階。
  Z折りは左右を同時に互い違いに回転。二つ折りは表紙でない側を背面へ折る。
  紙の厚み分、180°より僅かに手前で止めて重なりを表現。
- デザイン画像は「1面=1パネルに1データ」。アップロード済みパネルは画像を全面に1枚マッピング
  (`repeat=1, offset=0`)。未アップロードのパネルは Canvas 生成のガイド画像を
  `texture.repeat / offset` でパネル数ぶんに等分スライスして表示(パネル番号・折り線入り)。
  中面(裏面)は裏から見るとパネル順が左右逆になるため、ガイドのオフセットを反転して割り当てる。
- 各スロットは `dragover/drop` で D&D、`<label>` 内の隠し `input[type=file]` でクリック選択に対応。
  折り方変更時は `renderSlots()` がスロットを再生成し、設定済み画像(`panelTex`)は引き継がれる。
  設定済みスロットの×ボタンは `deletePanel()` で画像を破棄しガイドへ戻す。
- 両面プルーフは `renderProof()` が生成。展開(`fold=0`)・白背景・均一照明にして
  表面を正面(`theta=0`)・中面を背面(`theta=π`)から平行に近い向きで撮影し、1枚のPNGに上下合成する。
  WebGL のフレームバッファ読み出しのため `WebGLRenderer({ preserveDrawingBuffer: true })` を有効化。
- 右下「この画像でパンフレットを依頼する」ボタンは依頼フォーム(モーダル)を開く。送信は
  [FormSubmit](https://formsubmit.co/) を利用し、バックエンド無しで指定メールへ通知する。
  - フォームは `enctype="multipart/form-data"` の通常POST。`submit` 時に `renderProof('image/jpeg')` の
    結果を `dataURLtoFile()` で File 化し、`DataTransfer` で隠し `input[name=attachment]` に注入して自動添付する。
  - 制御用の隠しフィールド: `_subject` / `_template=table` / `_captcha=false` / `_autoresponse` / `_next`。
    `_next` は http/https で表示中のときのみ現ページURLを入れ、送信後に戻る。
  - **通知先メールはフォームの `action="https://formsubmit.co/<メール>"` で指定**(現状 `issei.masuya@onebe-create.com`)。
  - **初回のみ FormSubmit からのアクティベーション必須**: 公開後に一度テスト送信し、担当者メールに届く
    確認リンクをクリックするまで通知は配信されない。
  - 添付サイズ対策として画像は JPEG(品質0.92)で送信。WebGL 読み出しのため `preserveDrawingBuffer: true`。
  - 注意: `file://` で開いた場合も送信自体は可能だが `_next` での復帰は効かない(FormSubmit の既定ページが表示)。
    実運用は静的ホスティング(http/https)推奨。

## 免責

本ツールのシミュレーション結果はイメージです。実際の印刷物の仕上がり
(色味・折り位置・紙厚など)を保証するものではありません。
